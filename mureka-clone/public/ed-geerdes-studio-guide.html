ED-GEERDES — Studio guide

This file lives in public/ and is copied into production builds (Vercel, Docker static, etc.). No network required to read it after deploy.

Keep a copy: use your browser’s File → Save Page As… (or Print → Save as PDF). Same content ships at /ed-geerdes-studio-guide.html on your host.

Time & session

The React app header shows local wall clock and a session timer (time since this browser tab loaded). Refresh resets the session — use it as a light sense of how long you’ve been working. Audio players show elapsed · duration under Play / Pause / Stop.

Playback — Play, Pause, Stop

Play starts or resumes. Pause freezes at the current position. Stop pauses and jumps back to the start (0:00). The browser’s native audio bar is still there for scrubbing and volume on most screens.

Your own gateway (auto-connect)

Open API keys & sync in the sidebar (Connections modal). Use My gateway URL with your full API base, e.g.:

https://your-service.up.railway.app/api

Check Always connect with this gateway in this browser and Save. On the next full page load, that URL wins before other saved bases — no rebuild needed.

Optional build-time default (one URL baked into the bundle for teams): set VITE_MY_GATEWAY_URL in .env / hosting env, then rebuild. Priority order is documented in src/dieterClientConfig.js.

Voice studio · saves

Successful Mureka renders are listed under Your library · this browser (localStorage). Use Full song on Mureka with lyrics + API key to test without uploading a beat; use Mix: Mureka vocals + your beat when the backend exposes /api/pure-song-mureka.

Cover lab

Instrument cover keeps timing from your stem. Defaults: cover layer about −6 dB, blend 30% with the original — adjust before generating. Generation uses POST /api/cover/generate (multipart file).
